# Contributing to Universal Groovebox Converters (UGC)

This project is very much a work in progress, and contributions are always welcome.

## How to Contribute

There are several ways you can contribute to this project:

### 1. File an Issue

If you find a bug, have a feature request, or just want to suggest an improvement, please feel free to open an **issue** on the GitHub repository. Provide as much detail as possible, including:
- A clear and descriptive title.
- Steps to reproduce the issue (if it's a bug).
- Any relevant files or logs.

I can't promise that I can fix them swiftly, but I can always refer to it as a prioritized backlog. Even better:


### 2. Submit a Pull Request

If you want to contribute code, follow these steps:

1. Fork the repository.
2. Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Implement your changes and verify them using the files in the `testing/` directory.
4. Commit your work with a concise, descriptive message (squashing into a single commit is preferred).
5. Push the branch to your fork:
   ```bash
   git push origin feature/my-awesome-feature
   ```
6. Open a **Pull Request (PR)** against the main branch of this repository.

I will review the PR as soon as time and energy allow. 


### 3. Share Knowledge

This project involves reverse-engineering and understanding various binary formats (SP404mk2, M8, MPC, etc.). If you have insights, documentation, or sample files that could help, please share them via issues or discussions!


## Getting Started

To set up the project locally for development, please refer to the `README.md` for instructions on installing dependencies and running the tools.
