## 2025-11-10 SML big endian to little endian 16 bit conversion (gemini)

This script assumes a single-channel (mono) audio track with a 44100 Hz sample rate. You can adjust these parameters as needed.

import numpy as np
import wave
import os

def convert_raw_be16_to_wav(input_file, output_file, sample_rate=44100, n_channels=1):
    """
    Converts a raw 16-bit big-endian PCM audio file to a standard WAV file.

    Args:
        input_file (str): Path to the input raw audio file (16-bit big-endian).
        output_file (str): Path for the output WAV file.
        sample_rate (int): The sample rate of the audio data (e.g., 44100 Hz).
        n_channels (int): The number of audio channels (e.g., 1 for mono, 2 for stereo).
    """
    try:
        # 1. Read the raw binary data
        with open(input_file, 'rb') as f:
            raw_data = f.read()

        # 2. Convert raw bytes to a NumPy array of 16-bit integers
        # 'S' is used to tell numpy that the input data is a byte string (raw_data)
        # dtype='>i2' specifies:
        #   '>'  for big-endian byte order (the source format)
        #   'i2' for 16-bit signed integer (2 bytes)
        audio_array = np.frombuffer(raw_data, dtype='>i2')

        # 3. Perform byte-swap (Big-Endian to Little-Endian)
        # WAV files typically use little-endian byte order.
        # The .newbyteorder('<') method converts it in-place to little-endian ('<').
        # Using a copy to ensure the byte order change is applied correctly for the next step.
        little_endian_audio = audio_array.newbyteorder('<').copy()

        # 4. Convert the NumPy array back to bytes for the WAV file
        wav_frames = little_endian_audio.tobytes()

        # 5. Write the WAV file with the proper header
        with wave.open(output_file, 'wb') as wf:
            wf.setnchannels(n_channels)
            # The sample width is 2 bytes for 16-bit audio
            wf.setsampwidth(2)
            wf.setframerate(sample_rate)
            # Write the byte-swapped data
            wf.writeframes(wav_frames)

        print(f"Successfully converted '{input_file}' to '{output_file}'.")

    except FileNotFoundError:
        print(f"Error: Input file '{input_file}' not found.")
    except Exception as e:
        print(f"An error occurred during conversion: {e}")

⚙️ Key Concepts Explained
numpy.frombuffer(raw_data, dtype='>i2'):
This is the critical step for reading your raw data.
raw_data: The raw bytes read from your file.
dtype='>i2': This tells NumPy how to interpret the bytes:
>: Specifies Big-Endian byte order (your source format).
i2: Specifies a 16-bit (2-byte) signed integer.
.newbyteorder('<').copy():
WAV files (specifically the PCM data chunk) standardly use Little-Endian byte order.
The newbyteorder('<') method efficiently creates a view of the array data with the byte order swapped to Little-Endian (<).
.copy() is used to ensure we have a contiguous block of memory with the correct byte order before converting back to bytes.
wave Module:
This built-in Python package handles the complexities of generating the correct WAV header (RIFF, fmt, and data chunks).
wf.setsampwidth(2): Sets the bit depth to 16-bit (2 bytes).
wf.setframerate(SAMPLE_RATE): Sets the crucial sampling frequency.
wf.writeframes(wav_frames): Writes the final, little-endian audio data along with the calculated header.
If you don't know the exact sample rate or channel count of your raw data, you will need to determine those parameters before running the script, as the WAV header must contain that information.
